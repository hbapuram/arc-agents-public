The Problem & Solution

You have health data scattered across 5+ apps: sleep, cycle, meals, workouts, symptoms. None talk to each other. All advice is generic.

Arc Agents solves this by synthesizing everything together.

🔄 Synthesis, Not Collection

Most apps collect data and show dashboards. Arc reads all your data, understands the connections, and gives you one coherent recommendation.

🧠 Multi-Agent Intelligence

15 specialized agents (nutrition, fitness, cycle, sleep, etc.) all read the same health snapshot and coordinate seamlessly.

🔒 Privacy by Design

All data stays in your browser. No user database. No servers. No tracking. Completely offline-first.

📊 Real Personalization

Arc learns as you use it. The more you log, the better it understands your unique patterns.

How It's Built

Arc is a sophisticated system, but built pragmatically with real constraints in mind.

System Architecture

🖥️ Frontend (Browser)
React 18 SPA (no build step), 16 localStorage stores, token tracking, UI for all features
🌐 API Layer (Flask on Vercel)
Stateless serverless function. Routes: /chat, /context, /synthesize, /parse-entry, /macros, /parse-photo, etc.
🤖 AI Layer (Claude Haiku)
Fast, cheap, capable. System prompts specialized per agent. Token tracking on every call.
💾 Data Layer (localStorage)
16 stores: pantry, meals, recipes, schedule, chores, supplements, workouts, health log, symptoms, blood tests, body, water, shops, ingredient intel, sync status, tokens

Data Flow

When you ask an agent a question:

You ask (e.g., "What should I eat?")

Key Design Decisions

No Build Step

React UMD + Babel Standalone. Edit JSX, refresh browser, see change. Zero webpack complexity.

localStorage-First

5MB is a feature, not a limitation. Forces lean design. Caps on stores (500 meals max) prevent bloat.

Stateless API

Flask function routes to Claude. No persistent state. All compute in Claude, all data in browser.

Shared Context

All agents read the same "snapshot"—your health data in plain English. Enables coordination without explicit communication.

The 15 Agents

Each agent is specialized but receives the same health snapshot. This enables coordination without daisy-chaining.

The Integration Story

This is what makes Arc different: agents aren't daisy-chained or competing. They coordinate through shared context.

The "Shared Snapshot" Pattern

buildArcSnapshot() reads all 16 stores: { today: "2026-06-15", phase: "Luteal", sleep: { duration: 7, hrv: 55, quality: "good" }, meals: [ { name: "oatmeal", macros: {...} }, ... ], workouts: [ { type: "HIIT", duration: 30, feedback: "🟢" }, ... ], supplements: [ { name: "Magnesium", taken: true, phase: "Luteal" }, ... ], symptoms: [ { severity: 3, name: "bloating", day: 21 }, ... ], water: { logged: 1800, goal: 2000 }, ... } This snapshot is serialized as plain text and becomes part of EVERY agent's context.

Example: The Multi-Agent Conversation

You Ask Nora

"What should I eat today?"

Nora reads the snapshot and knows: Luteal phase (high appetite), HRV low (prioritize carbs), workouts this week = 12 sessions (protein high). Recommends more carbs than usual.

You Ask Felix

"Should I do HIIT today?"

Felix reads the snapshot and knows: HRV 55 (below peak), Luteal phase (lower tolerance), sleep 7h (adequate). Recommends gentle yoga instead.

They Never Conflict

Both see the same data. Both prioritize recovery. Both recommend the same thing without talking to each other—because they read the same snapshot.

Aurora Synthesizes

Morning brief: "Rest day today (HRV low). Eat more carbs (Luteal + low HRV). Yoga + meal prep. Magnesium with dinner."

All 5 recommendations are cohesive, not conflicting.

The Data Flow on Every Interaction

1. User logs data (meal, workout, symptom, etc.) ↓ 2. Data stored in localStorage (arc-[domain]-v1) ↓ 3. Arc fires "arc-[domain]-change" event ↓ 4. UI re-renders (instant feedback) ↓ 5. buildArcSnapshot() updates the in-memory health picture ↓ 6. When user chats with an agent: - Fetch /api/context/[agent-id] (system prompt + domain context) - Build client_context from relevant stores - Call /api/chat with full snapshot in context - Claude sees: system prompt + snapshot + client context - Returns recommendation - Token usage tracked locally

Why These Choices?

React UMD + Babel

No build step. Edit JSX, refresh browser. Fast development loop. Smaller cognitive load than webpack complexity.

localStorage as Backend

Privacy by default. 5MB limit forces lean design. No user database means no data at rest to leak. Fast reads/writes.

Stateless Flask

Scales to zero. Only pay for compute when in use. All state lives in browser. Easier to audit (code is simple).

Claude Haiku

Fast + cheap. Fast enough for real-time chat (< 1 sec), cheap enough for daily synthesis calls, capable enough for domain reasoning.

See It In Action

Here's how to use Arc Agents with real example data.

Step 1: Load Demo Data

When you first open Arc, go to the Settings tab and click "Load Demo Data". This populates the app with realistic example data so you can see how everything works.

📱 Settings Tab ├── Load Demo Data (button) └── [Click → Loads 7 days of meals, workouts, sleep, symptoms, etc.]

Step 2: Explore the Today Tab

The Today tab is your dashboard. You'll see:

🌅 Aurora Brief

Morning synthesis: 5 personalized actions for today based on your cycle phase, sleep quality, and workouts.

📊 Health Cards

Sleep (7h), HRV (55), Steps (12,000), Cycle phase (Luteal), Water intake (1800ml).

⏰ Schedule

Your day's plan: workouts, meals, supplements, care routines, chores.

💧 Water Tracker

Quick-add buttons: Glass, Mug, Bottle. Progress bar shows 1800/2000ml logged.

Step 3: Chat With Agents

Click on an agent to chat. Each agent sees your full health snapshot and gives personalized advice:

🍽️ Nora (Nutrition)

You: "What should I eat today?" Nora: "You're in your Luteal phase with low HRV (55). Today I recommend: - Carbs: 45% (from 40%) — progesterone increases appetite - Protein: 100g (stable) - Meals: Warm carbs, anti-inflammatory foods Try: Sweet potato + black beans + olive oil (fits your pantry) Macros: 650 kcal, 95g carbs, 18g protein, 22g fat"

Notice: Nora knew your phase, HRV, pantry, and macro goals—all without you telling her.

💪 Felix (Fitness)

You: "Should I do HIIT today?" Felix: "Not today. Your HRV is low (55) and you're in Luteal phase. Your body needs recovery. I recommend: - Yoga or gentle walk (30 min) - Prioritize sleep tonight - Save HIIT for your Follicular phase (higher tolerance) You did 3 workouts this week—good volume. Rest is the right call."

Felix coordinated with Nora: both recommend recovery without conflicting.

Step 4: Log Your Data

Go to Health → Data to log your health information:
